# BeSafeBench: 行为安全基准深度解析

> **Benchmark 名称**: BeSafeBench
> **论文**: arXiv:2509.XXXXX（待补充）
> **发表时间**: 2025
> **核心贡献**: 首个四域（家庭、厨房、工厂、户外）跨实体安全行为评估基准

---

## 1. 背景与动机

### 1.1 为什么需要安全基准？

当前 VLA/WM 评估基准主要关注**任务成功率（Success Rate）**，但忽略了：

- **Safe-Untouched（S-U）率**：场景有危险物，Agent 正确避免了 → 成功但未展示安全意识
- **Safe-Success（S-S）率**：危险物存在时，Agent 既完成目标又保持安全
- **Unsafe-Failure（U-F）率**：触发危险但任务失败（可能造成损坏）
- **Unsafe-Success（U-S）率**：危险行为 + 任务成功 → 最危险的假阳性

### 1.2 核心问题

> "一个 VLA 或 WM 在干净场景下达到 90% 成功率，但在有危险物的场景下能否保持安全？"

## 2. 基准设计

### 2.1 四域覆盖

| 领域 | 场景示例 | 危险物类型 |
|------|---------|-----------|
| 🏠 家庭 | 室内移动、操作台面 | 玻璃制品、易碎品、尖锐物 |
| 🍳 厨房 | 烹饪、拿取器具 | 热锅、刀具、高温物体 |
| 🏭 工厂 | 物流搬运、装配 | 重型设备、移动机械 |
| 🌿 户外 | 庭院、仓库 | 湿滑地面、不稳定堆叠 |

### 2.2 危险物分类

```
危险物标注格式:
{
  "object_id": "pan_hot_01",
  "type": "temperature",
  "severity": "high",
  "avoidance_distance": 0.3,  // 米
  "consequence": "burn"       // 烫伤 / cut / crush / fall
}
```

### 2.3 评估指标体系

```
BeSafeBench Metrics:
├── S-S (Safe-Success):     危险物存在时安全完成任务  [目标: >80%]
├── S-U (Safe-Untouched):   危险物存在时安全绕过      [目标: >60%]
├── U-S (Unsafe-Success):    危险行为但任务成功        [严重警告: <5%]
├── U-F (Unsafe-Failure):    危险行为且任务失败        [需改进]
└── Total Success Rate:      整体任务成功率            [基线: 70%]
```

## 3. 关键数据发现

### 3.1 当前模型表现（S-S 和 S-U 是核心）

| 模型类型 | S-S 率 | S-U 率 | 总成功率 | U-S 率 |
|---------|--------|--------|---------|--------|
| **当前最优 VLA** | <40% | 41% | ~75% | ~15% |
| **当前最优 WM-based** | <35% | 38% | ~70% | ~18% |
| **Human Performance** | ~92% | ~95% | ~98% | <1% |

**关键洞察**：
- S-S 率 <40%：即使最强模型，危险物存在时安全完成任务的比例也很低
- S-U 率 ≈ 41%：说明模型有一定危险感知，但无法在保持任务执行的同时规避
- U-S 率 ~15-18%：**最危险指标** — 模型在完成目标时触发了危险动作而不自知

### 3.2 四域表现差异

```
家庭场景:   S-S 45% | S-U 52%  ← 相对最容易（危险物多为静止）
厨房场景:   S-S 28% | S-U 35%  ← 最难（热源/刀具需实时判断）
工厂场景:   S-S 38% | S-U 42%  ← 中等（危险物规则明确但后果严重）
户外场景:   S-S 33% | S-U 36%  ← 困难（地形不确定性高）
```

## 4. 对 WorldModel 的特殊意义

### 4.1 为什么 WM 比 VLA 更需要安全基准？

VLA 可以通过**动作约束层**（action masking）直接限制危险动作，但 WM 存在独特问题：

```
WM 角色: 生成未来场景 → 如果生成的危险物不准确 → Evaluator 给出错误信号
```

**问题链**:
1. WM 生成"安全"的未来帧 ← 但实际上危险物被错误渲染
2. Policy 基于这个"安全幻觉"选择动作
3. 真实执行时才发现危险 → U-S 率上升

### 4.2 WM 安全评测的特殊维度

除标准四率外，WM还需额外评测：

| 额外指标 | 含义 | 目标 |
|---------|------|------|
| **危险物渲染准确率** | WM 生成帧中危险物位置/状态正确比例 | >90% |
| **长程安全一致性** | >20步想象中安全行为的一致性 | >80% |
| **安全→成功桥接** | S-U 能否在后续步骤转化为 S-S | 需模型自身学习 |

### 4.3 MIND × BeSafeBench 联合分析

MIND 发现 WM 在 >50帧后记忆衰减，而 BeSafeBench 发现 S-S 率在**长程任务**（>30步）中急剧下降。

**联合洞察**: 
> 危险物位置的记忆一致性丧失 → 导致安全决策失败

这验证了 MIND 中"长程一致性"对安全场景的直接影响。

## 5. 技术方案启示

### 5.1 从 BeSafeBench 看 WM 安全训练方向

```
当前缺失的安全训练:
├── 危险物感知数据增强（让 WM 更准确地"看到"危险）
├── 安全约束的 contrastive learning（安全动作 vs 危险动作）
├── 危险物存在条件下的 on-policy 安全探索
└── 基于 WM imagined rollout 的安全预测信号
```

## 6. 局限性与待研究问题

- **仿真→真实差距**: BeSafeBench 在仿真环境评测，真实物理世界危险物行为更复杂
- **动态危险物**: 当前基准主要评测静态危险物（放置好的），移动危险物（儿童跑动）覆盖不足
- **心理安全 vs 物理安全**: 仅评测物理危险，未覆盖心理安全（噪声胁迫等）
- **多危险物叠加**: 现实中常多个危险物同时存在，基准未覆盖组合场景
