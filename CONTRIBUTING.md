# CONTRIBUTING.md — WM-Research 贡献指南

> 欢迎为 WM-Research 知识库做出贡献！本文档定义了贡献规范和质量标准。

---

## 🎯 贡献类型

### 📝 论文解析
- **深度解析**（≥1KB）：论文动机 + 核心创新 + 关键数据 + 工程细节 + 消融实验
- **摘要级**（<500字）：新增论文的简短描述

### 🔬 研究空白分析
- 新兴研究空白识别
- 现有空白的重要更新
- 研究机会的量化评估

### 🐛 错误修正
- 事实性错误
- 链接失效
- 格式问题

### 📊 社区实战笔记
- GitHub Issues 高价值发现
- 小红书/英文社区经验
- 硬件/训练实战数据

---

## 📋 写作标准

### 深度解析模板
```markdown
# [论文名称]: [一句话描述]

## 🔑 一句话总结
...

## 📌 核心定位
### 解决的问题
### 关键创新

## 🏗️ 架构设计
...

## 📊 关键实验结果
...

## 💡 研究空白与机会
...

## 🔗 参考链接
```

### 论文索引格式
```markdown
| 论文 | 方向 | 链接 | 状态 |
|------|------|------|------|
| xxx | xxx | path | ✅/🔄/⬜ |
```

---

## ✅ 提交流程

1. **Fork 本仓库**
2. **创建分支**: `git checkout -b feat/paper-name`
3. **提交内容**: 确保符合上述格式标准
4. **发起 PR**: 描述改动内容和理由
5. **等待审核**: 子修/OPC-WM Agent 将在 24h 内审核

---

## 🚫 不接受的内容

- 纯摘要复制（无原创分析）
- 未经核实的事实
- 与现有内容完全重复
- 营销/软文性质内容

---

## 📊 质量门禁

| 类型 | 最低要求 |
|------|---------|
| 深度解析 | ≥1KB，包含关键数据和工程细节 |
| 摘要 | ≥200字，包含核心创新点 |
| 论文索引 | 包含 arXiv 链接或 GitHub 地址 |

---

_感谢你的贡献！_
